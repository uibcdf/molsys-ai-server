
# ADR-008: Entorno de desarrollo Python y tooling

## Fecha
2025-12-04

## Estado
Aceptada

## Contexto
MolSys-AI requiere una base de desarrollo limpia, moderna y mantenible.
El proyecto se escribirá mayoritariamente en Python.

## Decisión
- Versión mínima de Python: **3.10**
- Herramientas:
  - **ruff** (linter + formateador)
  - **pytest** para tests
  - **mypy** para type checking opcional
  - **uv** o **pip** como gestor de entorno (a elección del usuario)
- Estándar de packaging: `pyproject.toml` con `setuptools` simple.

## Opciones consideradas
### 1. ruff + pytest + mypy (elegida)
**Pros**
- Rápido y moderno.
- `ruff` reemplaza a flake8/isort/black.
- Muy buena experiencia de desarrollo.

**Contras**
- Requiere aprender ruff config (mínimo).

### 2. black + flake8 + isort
**Pros**
- Muy conocido.

**Contras**
- Más lento.
- Menos integración.

### 3. Poetry como gestor
**Pros**
- Muy cómodo.

**Contras**
- Sobrecarga para un proyecto multi-componente.

## Consecuencias
- Proyecto coherente con el ecosistema científico moderno.
- Facilita mantener la calidad del código.

## Notas futuras
- Se evaluará integrar `pre-commit` hooks cuando madure el proyecto.
