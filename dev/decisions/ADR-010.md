
# ADR-010: Seguridad y alcance de las tools del agente

## Fecha
2025-12-04

## Estado
Propuesta

## Contexto
MolSys-AI ejecutará herramientas que manipulan sistemas moleculares,
cargan archivos y podrían en el futuro lanzar simulaciones.

Es necesario limitar qué puede hacer el agente.

## Decisión
Se definen dos modos:

### 1. Modo seguro (por defecto)
- El agente **NO** puede:
  - ejecutar comandos de shell,
  - escribir fuera de un directorio sandbox,
  - borrar archivos,
  - acceder a red externa.
- Solo puede llamar tools explícitas registradas.

### 2. Modo avanzado (opt-in)
- Permitirá:
  - ejecutar scripts de análisis,
  - generar archivos de reporte,
  - lanzar workflows controlados.

## Justificación
Previene acciones destructivas o peligrosas.
La mayoría de usuarios científicos no necesita superpoderes peligrosos.

## Consecuencias
- Las tools deben ser **idempotentes** y **deterministas**.
- El executor debe validar todas las entradas.

## Notas futuras
- Definir auditoría y logging estructurado de las tool-calls.
